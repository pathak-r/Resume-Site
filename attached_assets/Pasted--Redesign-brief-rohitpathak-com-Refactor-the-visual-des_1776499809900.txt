# Redesign brief — rohitpathak.com

Refactor the visual design of the existing site to the "catalog" style described below. **Do not change the site's structure, routing, sections, or content.** Keep all existing copy except where this brief explicitly rewrites it. The goal is purely a visual + typographic overhaul.

---

## 1. Design philosophy

The new look is **editorial / archival** — think a well-kept index or library catalog. It is quiet, typographic, and confident. It is *not* flashy, gradient-heavy, or pill-button-driven.

Explicit "don'ts" (remove these from the current site):
- No gradient blobs, no colored glow behind cards
- No blue pill badges for categories (e.g. "GENERATIVE AI PRODUCT")
- No red accent text for impact lines (e.g. "3× FASTER ENGINEERING WORKFLOWS")
- No red/blue dual-accent system
- No large rounded pill buttons
- No colored top-edge accent strips on cards

---

## 2. Layout system

Every major section uses the same two-column grid:

```
| 140px index column | 1fr content column |
```

- **Left (index) column** contains: entry number (e.g. "№ 02"), a large numeral (e.g. "02"), and a small category label (e.g. "PROPTECH", "VISION AI", "ABOUT").
- **Right (content) column** contains: a small "Filed under" or "Project" label, the heading, body copy, and tag row.
- Gap between columns: `2rem`.
- Section separators: a single `0.5px` horizontal rule in a very light neutral.

On viewports under 720px, collapse to a single column — the index row sits above the content, inline and smaller.

---

## 3. Typography

- **Body / UI**: keep the current sans-serif (Inter or similar system sans is fine).
- **Headings**: same family, weight **500 only** — never 600 or 700. Letter-spacing `-0.02em` on the hero heading.
- **Case**: sentence case everywhere. No Title Case, no ALL CAPS except for the small index labels and section eyebrows.
- **Sizes**:
  - Hero H1: 36–44px, weight 500, line-height 1.15
  - Section H2: 24–28px, weight 500
  - Project H2: 22px, weight 500
  - Body: 14–15px, line-height 1.75
  - Eyebrow / index labels: 11px, letter-spacing 0.12em, uppercase
  - Large entry numerals: 40–56px, weight 500, line-height 1

---

## 4. Color

Replace the current blue (#0EA5E9-ish) + red (#DC2626-ish) dual-accent with a **single quiet palette**:

- Background (page): `#F4F2EC` or a similar warm off-white (the current `--color-background-secondary` tone is close)
- Background (cards / inner surfaces where needed): `#FFFFFF`
- Text primary: near-black, e.g. `#1A1A1A`
- Text secondary: `#5F5E5A`
- Text tertiary (labels, metadata): `#888780`
- Borders / rules: `rgba(0,0,0,0.12)` — always 0.5px
- **Optional single accent** (use sparingly, if at all — for the TestFlight badge or similar): a deep ink-teal `#0F6E56`. Do not use red. Do not use the current bright blue.

Dark mode: not required, but if implemented, invert cleanly (dark neutral bg, light text, same hierarchy).

---

## 5. Components

### Buttons
Replace all current pill buttons with **outline buttons**:
- `border: 0.5px solid` (primary button uses solid `#1A1A1A` border, secondary uses the light neutral border)
- `border-radius: 8px` (not fully rounded)
- `padding: 6px 14px`
- `font-size: 13px`
- Text only, no background fill. Primary CTA has a subtle bg on hover.

### Tags (tech stack, expertise)
- Outline chips, `border: 0.5px solid` light neutral
- `border-radius: 8px`, `padding: 3px 10px`
- `font-size: 11px`, text in secondary color
- **Do not** fill them with gray as the current site does

### Cards
Most sections should **not** use cards at all — just section separators. Use a card (white bg, 0.5px border, 12px radius) only for the Education block and Contact blocks.

## 7. Implementation notes for the agent

- Keep the existing React/Next.js (or whatever framework Replit built it in) component structure. This is a restyle, not a rewrite.
- Centralize the design tokens (colors, spacing, type scale) in a single theme file or CSS variables block before touching components.
- Build one reusable `<EntryRow>` component for the 140px | 1fr grid and use it for hero, every project, about, and experience. This is the single most important refactor.
- Remove any animation libraries or scroll-reveal effects. The design is still and confident; it should not fade or slide in.
